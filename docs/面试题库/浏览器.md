### Q2: 回流（Reflow）与重绘（Repaint）
> &#11088;&#11088;&#11088;&#11088;&#11088; | 公司：字节(2025一面)、腾讯(2024一面)、阿里(2025一面) | 难度：中级

**题目**：什么是回流和重绘？分别有哪些触发条件？如何减少回流和重绘？

**考察点**：
- 回流：布局改变，重新计算元素几何信息（尺寸、位置）
- 重绘：样式改变但布局不变（颜色、背景等）
- 回流一定触发重绘，重绘不一定触发回流
- 触发回流的操作：增删 DOM、修改尺寸/位置、`offsetTop` 等强制同步布局
- 优化：批量修改 DOM（`documentFragment`）、`class` 代替 `style`、`transform` 代替 `top/left`、`will-change` 创建合成层

> 答案参考：[../浏览器/reflow-repaint.md](../浏览器/reflow-repaint.md)

### Q4: XSS 和 CSRF
> &#11088;&#11088;&#11088;&#11088;&#11088; | 公司：所有公司必问（字节/阿里/腾讯/美团/京东/滴滴） | 难度：中级

**题目**：什么是 XSS 和 CSRF？分别怎么防御？

**考察点**：
- XSS 三种类型（反射型/存储型/DOM 型）+ 攻击原理
- CSRF 攻击原理（利用浏览器自动携带 Cookie 的特性）
- XSS 防御：HTML 实体转义 + CSP + HttpOnly Cookie + DOMPurify
- CSRF 防御：SameSite Cookie + CSRF Token + Referer/Origin 校验
- XSS vs CSRF 的本质区别（XSS 偷数据，CSRF 伪造操作）
- XSS 能绕过 CSRF Token

> 答案参考：[../浏览器/xss-csrf.md](../浏览器/xss-csrf.md)
> 延伸：[../浏览器/browser-security.md](../浏览器/browser-security.md)

### Q5: 浏览器缓存策略
> &#11088;&#11088;&#11088;&#11088; | 公司：阿里(2025一面)、字节(2024一面)、腾讯(2025一面) | 难度：中级

**题目**：强缓存和协商缓存有什么区别？Cache-Control 有哪些值？

**考察点**：
- 强缓存：`Cache-Control: max-age=3600` / `Expires`，状态码 200 (disk cache)
- 协商缓存：`ETag/If-None-Match` + `Last-Modified/If-Modified-Since`，状态码 304
- `Cache-Control` 指令：`max-age`/`no-cache`/`no-store`/`public`/`private`/`must-revalidate`
- 实际场景：`index.html` 用 `no-cache`（每次校验），静态资源用 `max-age=31536000`（文件名 hash）
- `ETag` 比 `Last-Modified` 更精确（文件内容 vs 修改时间）

> 答案参考：[../浏览器/cache.md](../浏览器/cache.md)

### Q6: Cookie 的 SameSite 属性
> &#11088;&#11088;&#11088;&#11088;&#11088; | 公司：阿里(2025一面)、字节(2024二面)、腾讯(2025一面) | 难度：中级

**题目**：SameSite 有哪些值？分别有什么安全影响？

**考察点**：
- `Strict`：任何跨站请求都不带 Cookie（最安全但体验差）
- `Lax`：浏览器默认值，跨站时只对"顶级导航 GET 请求"携带 Cookie
- `None`：允许跨站携带，必须配合 `Secure`（仅 HTTPS）
- HttpOnly（防 XSS）+ Secure（防 MITM）+ SameSite（防 CSRF）三层防御
- Cookie vs LocalStorage vs Token 的认证方案选型

> 答案参考：[../浏览器/cookie.md](../浏览器/cookie.md)
> 延伸：[../浏览器/storage.md](../浏览器/storage.md)

### Q8: Event Loop（事件循环）
> &#11088;&#11088;&#11088;&#11088;&#11088; | 公司：所有公司必问 | 难度：中级

**题目**：什么是事件循环？宏任务和微任务有什么区别？

**考察点**：
- 宏任务：`setTimeout`/`setInterval`/`I/O`/UI rendering/`MessageChannel`
- 微任务：`Promise.then`/`MutationObserver`/`queueMicrotask`
- 执行顺序：宏任务 → 所有微任务清空 → 渲染 → 下一个宏任务
- 经典题：`setTimeout` + `Promise` 混合输出顺序
- Node.js 的事件循环（6 个阶段，`process.nextTick`）

### Q9: requestAnimationFrame
> &#11088;&#11088;&#11088;&#11088; | 公司：字节(2025一面)、美团(2024一面) | 难度：中级

**题目**：requestAnimationFrame 和 setTimeout 做动画有什么区别？

**考察点**：
- rAF 在每帧渲染前执行（与屏幕刷新率同步 60fps）
- setTimeout 最小延迟 4ms（嵌套调用），可能在一帧之内或之外执行
- rAF 页面不可见时自动暂停（省电），setTimeout 继续执行
- requestIdleCallback 在帧末尾空闲时执行（低优先级任务）
- 具体时间线：每个帧内 rAF → Layout → Paint → requestIdleCallback

> 答案参考：[../浏览器/request-animation-frame.md](../浏览器/request-animation-frame.md)

### Q10: 多进程架构
> &#11088;&#11088;&#11088;&#11088; | 公司：字节(2025一面)、阿里(2024二面) | 难度：中级

**题目**：Chrome 浏览器有哪些进程？多进程架构有什么好处？

**考察点**：
- Browser 进程（主控）、Renderer 进程（渲染）、GPU 进程、Network 进程、Plugin 进程
- 每个 Tab 一个 Renderer 进程（Site Isolation 后同 site 的 iframe 也独立进程）
- 好处：一个 Tab 崩溃不影响其他 Tab、sandbox 安全隔离
- 缺点：内存开销大（Chrome 的内存"大"主要来自多进程）
- 与操作系统进程/线程模型的关系

> 答案参考：[../浏览器/browser-architecture.md](../浏览器/browser-architecture.md)

### Q11: 垃圾回收（GC）
> &#11088;&#11088;&#11088;&#11088; | 公司：字节(2025二面)、阿里(2024二面) | 难度：高级

**题目**：V8 的垃圾回收机制是怎样的？

**考察点**：
- 分代回收：新生代（Scavenge，空间换时间）+ 老生代（Mark-Sweep + Mark-Compact）
- 增量标记（Incremental Marking）：把一次 GC 拆成多段，减少主线程暂停时间
- 三色标记法：白（未访问）/灰（已访问，子节点未访问）/黑（已访问，子节点已访问）
- 引用计数 vs 标记清除
- WeakMap / WeakRef 弱引用在 GC 中的作用

### Q13: Service Worker 与 PWA
> &#11088;&#11088;&#11088;&#11088; | 公司：字节(2025二面)、美团(2024一面) | 难度：高级

**题目**：Service Worker 是什么？它的生命周期是怎样的？

**考察点**：

### Q15: 浏览器存储方案对比
> &#11088;&#11088;&#11088;&#11088; | 公司：腾讯(2025一面)、字节(2024一面) | 难度：初级

**题目**：Cookie、LocalStorage、SessionStorage、IndexedDB 有什么区别？

**考察点**：
- Cookie：4KB、自动携带到请求、可设 HttpOnly/过期时间
- LocalStorage：5MB、永久存储、同源共享
- SessionStorage：5MB、标签页关闭即删、标签页间不共享
- IndexedDB：无上限、异步、支持索引和事务、存储结构化数据
- 实际选择：Token 存内存（XSS 安全），Refresh Token 存 HttpOnly Cookie

> 答案参考：[../浏览器/storage.md](../浏览器/storage.md)
> 延伸：[../浏览器/cookie.md](../浏览器/cookie.md) [../浏览器/indexeddb.md](../浏览器/indexeddb.md)

### Q16: 页面生命周期事件
> &#11088;&#11088;&#11088;&#11088; | 公司：字节(2025一面)、阿里(2024一面) | 难度：中级

**题目**：DOMContentLoaded 和 load 有什么区别？beforeunload 和 unload 呢？

**考察点**：
- `DOMContentLoaded`：DOM 树构建完成 + defer 脚本执行完（图片/CSS 可能还在加载）
- `load`：所有资源（图片/字体/iframe）加载完毕
- `beforeunload`：用户即将离开，可弹确认框（不能做异步操作）
- `unload`：页面即将被销毁（已不推荐，会阻止 bfcache，用 `pagehide` 替代）
- `visibilitychange`：Tab 切后台/前台，用于暂停/恢复资源消耗
- bfcache（回退缓存）：`pageshow.persisted` 检测是否从缓存恢复

> 答案参考：[../浏览器/page-lifecycle.md](../浏览器/page-lifecycle.md)

### Q17: Observer API
> &#11088;&#11088;&#11088;&#11088; | 公司：字节(2025二面)、美团(2024二面) | 难度：中级

**题目**：IntersectionObserver 和 MutationObserver 分别是什么？怎么用？

**考察点**：
- MutationObserver：观察 DOM 变化（属性/子节点/文本），异步微任务批量回调
- IntersectionObserver：观察元素可见性，懒加载+曝光埋点+无限滚动
- ResizeObserver：观察元素尺寸变化（比 window.resize 精确），图表自适应
- PerformanceObserver：观察性能条目，Web Vitals 采集底层
- vs 传统方案的性能优势（异步+批量+不阻塞主线程）

> &#11088;&#11088;&#11088; | 公司：美团(2024一面)、京东(2025一面) | 难度：中级

**题目**：Web Storage、Web SQL、IndexedDB 分别是什么？

### Q20: BOM 和 DOM 的区别
> &#11088;&#11088;&#11088; | 公司：腾讯(2024一面)、美团(2025一面) | 难度：初级

**题目**：什么是 BOM？BOM 和 DOM 有什么区别？

**考察点**：
- DOM：文档对象模型（HTML/XML 文档的编程接口）→ `document`
- BOM：浏览器对象模型（浏览器提供的 JS 全局能力）→ `navigator`/`screen`/`location`/`history`
- DOM 有 W3C 规范标准，BOM 没有统一规范（各浏览器自行实现）
- `location.href` vs `location.replace` 的区别
- `URLSearchParams` 解析 URL 参数
- `navigator.onLine` 的局限性

> 答案参考：[../浏览器/bom.md](../浏览器/bom.md)
