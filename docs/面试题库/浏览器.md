### Q2: 回流（Reflow）与重绘（Repaint）
> ⭐⭐⭐⭐⭐ | 公司：字节(2025一面)、腾讯(2024一面)、阿里(2025一面) | 难度：中级

**题目**：什么是回流和重绘？分别有哪些触发场景？如何减少回流和重绘？

**考察点**：
- 回流：布局改变，重新计算元素几何信息（尺寸、位置）
- 重绘：样式改变但布局不变（颜色、背景等）
- 回流一定触发重绘，重绘不一定触发回流
- 触发回流的操作：增删 DOM、修改尺寸/位置、`offsetTop` 等强制同步布局
- 优化手段：批量修改 DOM（`documentFragment`）、`class` 代替 `style`、`transform` 代替 `top/left`、`will-change` 创建合成层

> 答案参考：[../浏览器/reflow-repaint.md](../浏览器/reflow-repaint.md)
> 延伸：[../浏览器/render-process.md](../浏览器/render-process.md)

### Q3: 强缓存与协商缓存
> ⭐⭐⭐⭐⭐ | 公司：腾讯(2024一面)、阿里(2025一面)、字节(2024二面) | 难度：中级

**题目**：请详细解释浏览器的强缓存和协商缓存机制。`Expires` / `Cache-Control` / `Last-Modified` / `ETag` 分别属于哪一类？各自优先级是怎样的？

**考察点**：
- 强缓存（不发送请求）：`Expires`（HTTP/1.0，绝对时间）和 `Cache-Control`（HTTP/1.1，相对时间 `max-age`）
- `Cache-Control` 指令：`no-cache`（协商缓存）/`no-store`（不缓存）/`public`/`private`
- 协商缓存（发送请求，304）：`Last-Modified` / `If-Modified-Since` 和 `ETag` / `If-None-Match`
- `ETag` 优先级高于 `Last-Modified`（秒级精度不足、文件内容不变但修改时间变）
- 实际部署策略：HTML 用协商缓存，JS/CSS/图片用强缓存 + hash 文件名

> 答案参考：[../浏览器/render-process.md](../浏览器/render-process.md)

### Q4: 浏览器 Event Loop 详解
> ⭐⭐⭐⭐⭐ | 公司：字节(2025一面)、腾讯(2024二面)、阿里(2025一面) | 难度：中级

**题目**：请描述浏览器的 Event Loop 机制，包括宏任务、微任务，以及 `requestAnimationFrame` 在其中的位置。以下代码输出什么？

```javascript
setTimeout(() => console.log('macro'), 0);
Promise.resolve().then(() => console.log('micro'));
requestAnimationFrame(() => console.log('rAF'));
```

**考察点**：
- 宏任务：`setTimeout`、`setInterval`、I/O、UI rendering
- 微任务：`Promise.then`、`MutationObserver`、`queueMicrotask`
- 一个宏任务 -> 清空微任务 -> （可选）渲染 -> 下一个宏任务
- `requestAnimationFrame` 在渲染之前执行
- 代码题输出：micro -> rAF -> macro（rAF 在第一帧渲染前）

> 答案参考：[../JavaScript/event-loop.md](../JavaScript/event-loop.md)
> 延伸：[../Node/node-event-loop.md](../Node/node-event-loop.md)

### Q5: 跨域解决方案
> ⭐⭐⭐⭐ | 公司：美团(2024一面)、字节(2025一面)、腾讯(2024一面) | 难度：中级

**题目**：浏览器的同源策略是什么？有哪些解决跨域问题的方案？各自优缺点是什么？

**考察点**：
- 同源策略：协议 + 域名 + 端口，三者都相同才同源
- CORS（跨域资源共享）：`Access-Control-Allow-Origin` 等服务端响应头
- JSONP：利用 `<script>` 标签不受同源限制，只能 GET，回调注入
- 代理（Proxy）：开发环境 devServer proxy，生产环境 Nginx 反向代理
- postMessage：iframe 跨域通信
- WebSocket：不受同源策略限制

> 答案参考：[../网络/http-https.md](../网络/http-https.md)
> 延伸：[../安全/csrf.md](../安全/csrf.md)

### Q6: Web Storage（localStorage / sessionStorage / cookie）
> ⭐⭐⭐⭐ | 公司：阿里(2025一面)、腾讯(2024一面)、美团(2024一面) | 难度：中级

**题目**：请对比 `cookie`、`localStorage`、`sessionStorage`、`IndexedDB` 的异同。各自适用于什么场景？如何选择合适的存储方案？

**考察点**：
- `cookie`：每次 HTTP 请求自动携带、4KB 限制、可设过期时间、可设 HttpOnly/Secure
- `localStorage`：持久化存储（不清除一直存在）、5MB、同源共享
- `sessionStorage`：页面会话期间有效、关闭标签页即清除、同源但不跨标签页
- `IndexedDB`：异步、支持事务、结构化数据、无大小限制
- 实际场景：Token 存 cookie（HttpOnly），用户偏好存 localStorage，临时数据存 sessionStorage

> 答案参考：[../浏览器/render-process.md](../浏览器/render-process.md)

### Q7: 进程与线程
> ⭐⭐⭐⭐ | 公司：腾讯(2024二面)、字节(2025一面)、阿里(2024二面) | 难度：中高级

**题目**：浏览器是多进程还是多线程？Chrome 的进程架构是怎样的？每个进程分别负责什么？

**考察点**：
- 浏览器是多进程架构（Chrome）：1 个 Browser 主进程 + N 个 Renderer 渲染进程 + 1 个 GPU 进程 + 1 个 Network 进程 + Plugin 进程
- 每个标签页一个渲染进程（站点隔离 Site Isolation）
- 渲染进程内部多线程：GUI 渲染线程、JS 引擎线程、事件触发线程、定时器线程、网络请求线程
- JS 引擎线程和 GUI 渲染线程互斥（JS 执行时暂停渲染）

> 答案参考：[../浏览器/render-process.md](../浏览器/render-process.md)

### Q8: DOMContentLoaded vs load
> ⭐⭐⭐⭐ | 公司：字节(2025一面)、腾讯(2024一面)、美团(2024一面) | 难度：中级

**题目**：`DOMContentLoaded` 和 `load` 事件有什么区别？`defer` 和 `async` 的 `<script>` 分别在哪个阶段执行？

**考察点**：
- `DOMContentLoaded`：HTML 解析完成、DOM 树构建完毕（图片/CSS 等资源可能还未加载完）
- `load`：页面所有资源（图片、CSS、JS、iframe）加载完毕
- `<script defer>`：下载不阻塞解析，在 `DOMContentLoaded` 之前按顺序执行
- `<script async>`：下载不阻塞解析，下载完立即执行（无序、可能在 `DOMContentLoaded` 前后）
- `<script>`：同步阻塞 HTML 解析

> 答案参考：[../浏览器/render-process.md](../浏览器/render-process.md)

### Q9: Web Worker
> ⭐⭐⭐ | 公司：美团(2025一面)、字节(2024二面) | 难度：中级

**题目**：Web Worker 解决了什么问题？它的通信方式是怎样的？有哪些限制？请简述项目中可能的使用场景。

**考察点**：
- 创建独立线程执行耗时 JS，不阻塞主线程 UI 渲染
- 通信方式：`postMessage` + `onmessage`，数据通过结构化克隆传递
- 限制：不能操作 DOM、不能访问 `window`/`document`、同源限制
- `SharedWorker`（多标签页共享）和 `Service Worker`（离线缓存/PWA）
- 场景：大数据处理、CSV/JSON 解析、加密运算、图像处理

> 答案参考：[../浏览器/render-process.md](../浏览器/render-process.md)

### Q11: 浏览器的合成层（Composite Layer）
> ⭐⭐⭐ | 公司：字节(2025二面)、腾讯(2024二面) | 难度：中高级

**题目**：什么是浏览器的合成层？什么操作会创建新的合成层？GPU 加速的优势和注意事项是什么？

**考察点**：
- 合成层的意义：将页面分层，在 GPU 上独立合成，不触发主线程重绘
- 创建合成层的方式：3D transform、`will-change`、`<video>`/`<canvas>`、opacity 动画
- 优势：高性能动画（transform + opacity 只用 compositor 线程）
- 注意事项：层爆炸（过多合成层导致内存占用高）、隐式合成
- `will-change` 的正确用法：动画前后动态添加/移除

> 答案参考：[../浏览器/reflow-repaint.md](../浏览器/reflow-repaint.md)

### Q12: 浏览器安全策略
> ⭐⭐⭐ | 公司：阿里(2025一面)、腾讯(2024面) | 难度：中级

**题目**：浏览器的安全策略有哪些？CSP（内容安全策略）、X-Frame-Options、SameSite Cookie 分别解决什么问题？

**考察点**：
- CSP 通过 HTTP 头限制资源来源（script-src/style-src 等），防御 XSS
- X-Frame-Options 防止页面被 `<iframe>` 嵌套（点击劫持）
- SameSite Cookie（Strict/Lax/None）防御 CSRF
- HTTPS 强制：HSTS（HTTP Strict Transport Security）
- CORB（Cross-Origin Read Blocking）防止跨域读取敏感数据

> 答案参考：[../安全/xss.md](../安全/xss.md)
> 延伸：[../安全/csrf.md](../安全/csrf.md)

### Q13: 页面生命周期事件
> ⭐⭐ | 公司：美团(2024面)、腾讯(2025一面) | 难度：中级

**题目**：`visibilitychange` 和 `pagehide` / `beforeunload` 有什么区别？如何监听用户是否离开了当前页面？

**考察点**：
- `visibilitychange`：页面可见性变化（切换标签页、最小化）
- `beforeunload`：页面即将卸载（刷新、关闭），可弹确认框
- `pagehide`：页面隐藏（包括移动端被系统杀死），`persisted` 属性判断 bfcache
- `pageshow`：页面显示（含从 bfcache 恢复），`persisted` 属性判断来源
- bfcache（往返缓存）的兼容处理：`pageshow` 恢复页面状态

> 答案参考：[../浏览器/render-process.md](../浏览器/render-process.md)

- 预加载扫描器（Preload Scanner）提前发现资源

### Q15: 前端监控中的性能指标
> ⭐⭐⭐ | 公司：字节(2025一面)、美团(2024二面) | 难度：中级

**题目**：前端页面性能的核心指标有哪些？FCP、LCP、FID、CLS、TTI、TBT 分别代表什么？如何通过 Performance API 获取？

**考察点**：
- FCP（First Contentful Paint）：首次内容绘制
- LCP（Largest Contentful Paint）：最大内容绘制（Core Web Vitals）
- FID（First Input Delay）：首次输入延迟，INP 正在替代 FID
- CLS（Cumulative Layout Shift）：累积布局偏移
- `PerformanceObserver` API 获取指标
- `performance.getEntriesByType('navigation')` 获取各阶段时间

> 答案参考：[../性能优化/web-vitals.md](../性能优化/web-vitals.md)
> 延伸：[../浏览器/render-process.md](../浏览器/render-process.md)
