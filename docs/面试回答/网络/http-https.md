# HTTP / HTTPS 面试回答

## Q1: HTTP 和 HTTPS 有什么区别？

### 30 秒版本

"HTTPS = HTTP + TLS 加密层。四个区别：HTTP 明文、HTTPS 加密；HTTP 80 端口、HTTPS 443；HTTPS 需要 CA 证书；HTTPS 防篡改和防窃听。"

### 2 分钟版本

"从用户访问 HTTPS 网址说起，浏览器和服务器做了三件事：

**证书验证**：服务器发来 SSL 证书——含公钥和 CA 签名。浏览器用内置根证书逐级验证证书链——根 CA→中间 CA→网站证书。任何一环不信任就报证书错误。

**密钥交换**：浏览器生成随机对称密钥，用服务器公钥加密后发给服务器。只有拥有私钥的服务器能解密拿到对称密钥。这一步是非对称加密（RSA/ECDHE）——慢但安全。

**对称加密通信**：之后全用对称密钥加密（AES）——快且安全。

**为什么不全用非对称加密？** 非对称比对称慢 100-1000 倍。用非对称交换对称密钥，是安全和性能的平衡。

**TLS 1.3 改进**：握手 2-RTT→1-RTT，去掉不安全加密算法，默认前向安全性。"

### 追问预判

| 面试官追问 | 你的回答 |
|-----------|---------|
| "中间人攻击怎么防" | HTTPS 证书链验证——攻击者没有 CA 私钥，无法伪造合法证书。前提是用户没手动信任不可信证书 |
| "HTTP2 相比 1.1 的改进" | 多路复用（一个连接并发多请求）、头部压缩（HPACK）、服务器推送、二进制分帧。最关键的是多路复用——解决 HTTP1.1 队头阻塞 |
| "HTTP 状态码分类" | 1xx 信息、2xx 成功(200/201/204)、3xx 重定向(301永久/302临时/307/308)、4xx 客户端错误(400/401未认证/403无权限/404/405)、5xx 服务端(500/502网关/504超时)。注意：304 是协商缓存状态码，属于 3xx 但不属于"重定向"——它告诉浏览器"用本地缓存，不用重新下载" |

## 别踩的坑

1. **"HTTPS = HTTP + SSL"太笼统** —— 面试官想听 TLS 握手和证书链。至少说清"非对称加密交换对称密钥"
2. **301 和 302 说反** —— 301 永久重定向（浏览器缓存，下次直接跳），302 临时（不缓存）
3. **304 和强缓存混为一谈** —— 304 是协商缓存的结果，有请求往返；强缓存无请求
