# 闭包 面试回答

## Q1: 什么是闭包？有什么用？

### 30 秒版本

"闭包就是函数能记住并访问它被创建时所在的作用域——即使这个作用域已经执行完了。最常见的例子是 `for + setTimeout` 输出题和模块化封装私有变量。闭包不是特殊语法，是 JS 作用域机制的自然结果。"

### 2 分钟版本

"理解闭包分两步——先懂作用域，再看引用关系。

**作用域是前提**：JS 是词法作用域——函数的作用域由定义位置决定，不是调用位置。你在 b 函数里调 a 函数，a 访问不到 b 的变量——因为 a 的词法作用域是它自己的定义位置。

**闭包是引用关系**：外层函数执行完后，作用域本应销毁。但内层函数被返回并在外部持有引用——内层函数还引用着外层函数的变量。于是这个作用域不能被 GC。**这就是闭包——函数 + 它能访问的外部变量 = 闭包。**

**实际场景**：
1. **数据私有化**：模块模式——IIFE 包住私有变量，返回公开 API。变量外部不可直接访问
2. **for + setTimeout 经典题**：`var` 是函数作用域，所有回调共享同一个 `i`（全 5）。`let` 是块作用域——每次迭代创建新的 `i` 绑定，每个回调闭包引用自己那个迭代的 `i`
3. **防抖节流的 timer**：timer 变量被返回的函数闭包持有——外部无法直接修改

**内存问题**：闭包会阻止变量被 GC——但这不叫内存泄漏。泄漏是你把大对象挂在闭包引用链上忘了释放。不需要时置 `null`。"

### 追问预判

| 面试官追问 | 你的回答 |
|-----------|---------|
| "闭包会导致内存泄漏吗" | 闭包本身不会——是你不小心把大对象存在闭包引用的变量里忘了清理。现代 JS 引擎只保留闭包中实际被访问的变量 |
| "let 为什么能解决 for+setTimeout" | let 块作用域——每次迭代创建新绑定，每个回调闭包引用独立的 `i` |
| "闭包与作用域链的关系" | 作用域链是查找变量的路径。闭包是内层函数"记住"了外层作用域——作用域链是实现闭包的机制 |

## 别踩的坑

1. **"闭包 = 函数里返回函数"** —— 形式不是本质。`setTimeout` 的回调引用外部变量也是闭包
2. **"闭包一定导致内存问题"** —— 闭包是 JS 的核心模式，不是 bug。只有引用了大对象又长期不释放时才可能有问题
3. **for + var + 异步** —— 面试必考题。`forEach` 每次迭代创建新闭包，`for` + `var` 不会
