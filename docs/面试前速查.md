# 面试前速查

> 这不是知识文档，是面试当天/前一晚的最后防线。每行一句话纠正一个致命错误——这些点说反了直接扣分。
>
> 原理和细节不在这里展开，记结论就行。

---

## JavaScript

- **旧**: async 函数 return 的 Promise 和原生 Promise.resolve 一样，都是一个微任务 tick → **正确**: async return 会多 2 个额外微任务 tick（thenable 采纳过程），所以输出顺序是 `1 3 4 5 2` 而不是 `1 3 4 2 5`
- **旧**: Proxy receiver 用普通方法就能演示 this 逃逸 → **正确**: 必须用 getter 属性才能演示——普通方法的 this 本来就会指向调用者（即 proxy），根本不存在"逃逸"
- **旧**: SharedArrayBuffer 和 ArrayBuffer 一样可以 transfer → **正确**: SharedArrayBuffer 是直接共享内存、不可 transfer，调用 `postMessage` transfer 会抛 DataCloneError；只有普通 ArrayBuffer 才能 transfer
- **旧**: WeakMap 的 key 只能是 Object → **正确**: ES2023 起 WeakMap key 也允许未注册 Symbol（`Symbol()` 创建的）

## Vue3

- **旧**: 父子挂载是先父 beforeMount → 父 mounted → 再子 beforeMount → 子 mounted → **正确**: 挂载顺序是父 setup → 父 beforeMount → 子 setup → 子 beforeMount → 子 mounted → 父 mounted，子 mounted 在父 mounted 之前
- **旧**: KeepAlive 是销毁组件再重建 → **正确**: KeepAlive 是把 DOM move 到隐藏的 storageContainer 里，不销毁、不走 mount 钩子
- **旧**: v-if 和 v-for 不能同时用是因为编译器会退化为非优化模式 → **正确**: 真正原因是 v-if 优先级比 v-for 高，编译时 v-if 拿不到 v-for 的 item 变量，而且无论如何都会遍历整个列表再判断，纯浪费
- **旧**: nextTick 用宏任务也能实现，只是 Vue 选了微任务 → **正确**: 如果 nextTick 用宏任务，浏览器会在回调前先渲染一帧旧状态，用户会看到闪烁——所以必须用微任务
- **旧**: SSR 渲染器和客户端 renderer 共享 patch 逻辑，只是换了 nodeOps → **正确**: SSR 渲染器走完全独立的字符串序列化路径（`renderToString`），跟客户端 renderer 不是同一套逻辑
- **旧**: `onBeforeUpdate` 在 scheduler 的 pre 队列里执行 → **正确**: `onBeforeUpdate` 不在任何队列里，它在组件 update job 内部、re-render 之前同步调用

## CSS

- **旧**: `@layer` 的 !important 规则也是后声明的层赢 → **正确**: !important 反转层叠优先级——先声明的层的 !important 反而赢，后声明的只对普通规则成立
- **旧**: `overflow: hidden` 和 `overflow: clip` 都会切断 sticky → **正确**: `overflow: clip` 不创建滚动容器，sticky 不受影响；它恰恰是用来替代 hidden 保住 sticky 的方案
- **旧**: 0.5px 在移动端可以放心用 → **正确**: 0.5px 在 Android 上因厂商而异，有些渲染成 1px、有些不显示，不可依赖

## 网络 / 浏览器

- **旧**: HTTPS 握手是浏览器生成会话密钥、用服务器公钥加密传过去 → **正确**: 现代 HTTPS 用 ECDHE，证书私钥只做签名认证，会话密钥双方各自独立算出、从不过网。RSA 密钥交换已废弃、TLS 1.3 已移除
- **旧**: Cookie `SameSite=Lax` 拦住所有跨站请求 → **正确**: Lax 放行顶级导航 GET（比如从邮件点链接跳转到你的网站），这是 CSRF 的剩余攻击面，不能全靠 SameSite
- **旧**: CSP `unsafe-eval` 是阻止 eval → **正确**: 方向完全反了——`unsafe-eval` 是**允许** eval，不加 `unsafe-eval` 才阻止
- **旧**: CORS Nginx 配置反射 `Origin` 头同时开启 `Access-Control-Allow-Credentials: true` 没问题 → **正确**: 这等于对任意网站开放凭证请求，绝对不能反射 Origin + 同时设 credentials

## 工程化 / Git

- **旧**: ESM 循环引用时，未初始化的导出变量是 `undefined` → **正确**: ESM 循环引用中，依赖模块先执行、被依赖模块后执行，访问未初始化的绑定直接抛 ReferenceError，不是 undefined
- **旧**: `git cherry-pick` 总会记录来源 commit → **正确**: 不加 `-x` 不会有任何来源信息，加了 `-x` 才会在 commit message 末尾追加 `(cherry picked from commit xxx)`
- **旧**: GitHub Actions artifact 用 v3 还能跑 → **正确**: artifact v3 已于 2025-01-30 关停，配置里必须用 `actions/upload-artifact@v4` 和 `actions/download-artifact@v4`

## Pinia

- **旧**: Pinia 保留 mutation 是因为 Vue3 `defineProperty` 无法追踪 `$patch` 之外的赋值 → **正确**: `defineProperty` 完全能追踪赋值——真实理由是 DevTools 快照可追踪 + 显式收敛修改入口，方便调试
- **旧**: Vuex 5 是 Pinia 之后的新版本 → **正确**: Vuex 5 从未发布，Pinia 就是 Vuex 5 RFC 的事实实现与官方继任者

## HR

- **旧**: 面试谈薪虚报当前薪资能抬 offer → **正确**: offer 前常核实工资流水和社保公积金基数，虚报穿帮直接出局，不要冒这个险
