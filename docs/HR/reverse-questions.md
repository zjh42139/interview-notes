# 反问面试官

> ⭐⭐⭐⭐⭐｜难度：初级｜这个环节没有标准答案，但有好答案和坏答案

## 一句话总结

**反问环节没有「不问」这个选项。问什么取决于对面是谁：技术面试官问技术决策和成长空间，HR 问薪资结构和晋升机制。永远不要问「公司是做什么的」——你应该在面试前就已经查过。**

## 按面试官类型

### 技术面试官（一面/二面技术）

**能问的**：

```
「你们团队的技术栈是怎么演进的？比如有没有从 Vue2 迁到 Vue3？」——了解团队技术现状
「团队的代码 review 流程是怎样的？新人上手大概多久能独立承担需求？」——了解融入过程
「你们最近攻克的最有挑战的技术问题是什么？」——了解团队技术深度，同时展示你的兴趣
「团队的技术分享文化怎么样？有没有跨组的技术交流？」——了解成长环境
```

**不能问的**：
- 「你们用什么技术栈」（面试前就该查）
- 「薪资怎么样」（跟技术面试官谈钱很不专业，薪资问 HR）
- 「加不加班」（暴露你在意这个）
- 「公积金交多少」（问 HR，不是问技术面试官）

### 业务/项目面试官（二面）

```
「这个岗位主要负责哪条业务线？目前这条线的技术挑战主要在哪个环节？」——了解你的工作内容
「团队的迭代周期大概是怎样的——是双周 sprint 还是有其他的节奏？」——了解工作方式
「目前团队里前后端的协作模式是怎样的？是并行开发还是后端先行？」——了解协作
```

### HR/终面面试官（三面/HR 面）

```
「公司的晋升机制是怎样的——技术序列是怎么分的，考核周期多久一次？」——展示你想长期发展
「试用期的期望是什么——转正的考核标准是怎样的？」——展示你想把事情做好
「公积金是按照全额工资交吗，基数是多少？」——财务问题，只能问 HR
「公司对于员工的技术成长有没有支持——比如会议赞助、学习经费？」——了解成长资源
```

## 按面试轮次

| 轮次 | 问什么 | 不问什么 |
|------|------|------|
| **一面**（技术面） | 团队技术栈、开发流程、技术挑战、code review 流程 | 薪资、福利、加班 |
| **二面**（项目/架构面） | 业务线情况、团队组成、前后端协作、迭代节奏 | 薪资、福利 |
| **三面**（终面/HR） | 晋升机制、薪资结构、公积金、转正标准、培训 | — |

## 别踩的坑

1. **「我没有问题」** —— 面试官会认为你对这个岗位没兴趣。至少准备 2-3 个问题问技术面试官、2 个问 HR。

2. **「公司是做什么的」** —— 面试前花 10 分钟看官网和最近新闻是最基本的功课。这个问题等于说「我懒得查」。

3. **只问福利不问工作** —— 前两面只问钱和福利会让面试官觉得你只在乎待遇。技术和业务面展示你对工作的兴趣。

4. **问面试官私人问题** —— 「你在这工作开心吗」「你工资多少」——很冒犯。

5. **一个问题问太多细节** —— 「晋升机制从 T6 到 T7 需要什么条件，大概多久，最近一次晋升比例是多少...」——展示了你做了功课但追问太多像在审问。点到为止，拿到核心信息即可。

## 面试追问

| 追问 | 回答 |
|------|------|
| 「你真的不想知道薪资吗」 | 「薪资我问 HR 就好——现在更想了解技术方面的事情」——加分回答 |
| 「你是在面试我们吗」 | 笑着回答「我确实想多了解一下团队——毕竟来这边工作是一个双向选择，我希望确认这是一个我能发光发热的地方」 |
