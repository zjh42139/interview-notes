# 阅读指南

> 分档逻辑：你的差异化优势是**项目实战 + Vue3/TS 技术栈深度理解**，不是「什么都懂一点」。以下分档从大厂 3 年前端面试官视角出发，按面试价值和 ROI 排序。

## ⭐ 跨模块知识链（P0 核心任务）

比再多几十篇文章价值高得多的是——**能把核心知识串成一条链**。每条链都应该能脱稿画出来，并说明相邻节点的因果关系。

### 链路 1：异步编程链

```
Vue Scheduler → Promise → Event Loop → 浏览器渲染 → Paint
```

为什么：Vue 的异步更新队列依赖 Promise 微任务 → 微任务在 Event Loop 中的位置决定了渲染时机 → 渲染前才执行 DOM 更新 → 减少回流次数。

### 链路 2：页面加载链

```
输入 URL → DNS → TCP → TLS → HTTP → HTML 解析 → CSSOM → Layout → Paint → 合成
```

为什么：这条链串起网络、浏览器、性能三个模块。面试中最常见的开放题就是"从输入 URL 到页面展示发生了什么"——你把这条链展开讲就是满分答案。

### 链路 3：构建优化链

```
Tree Shaking → ESM 静态分析 → Rollup → Vite → Code Splitting → 浏览器缓存
```

为什么：Tree Shaking 的前提是 ESM 静态结构 → 这决定了 Rollup/Vite 能做而 Webpack CJS 做不了的优化 → Code Splitting 把产物拆成多个 chunk → 配合强缓存实现长期缓存策略。

### 链路 4：安全链

```
同源策略 → CORS → CSP → XSS 防御 → CSRF 防御 → Token 存储 → Cookie SameSite
```

为什么：前端安全的所有问题都从同源策略出发——理解了同源就理解了跨域、XSS、CSRF、Cookie 安全。这条链的主线是"源 = 协议 + 域名 + 端口"。

### 练习方法

1. 拿出一张白纸，从链路起点开始画流向
2. 每一个箭头旁写一句因果：为什么这一步会走到下一步
3. 对比对应的知识文件，检查缺失的箭头
4. 重复直到不用看任何提示

## 执行建议

1. **先确认性质**：这些文件对你来说是「要复习的已学内容」还是「还要新学的内容」——两种情况的时间预算完全不同
2. **P0 反复过到能脱稿讲**，尤其是项目实战部分——这是你和 80% 候选人的分水岭。每读完一篇，关掉页面，30 秒内口头说清核心逻辑
3. **P1 理解到位就切下一个点**，不要在细节上停留太久
4. **P2/P3 不建议投入完整学习时间**，遇到面试官问到，用你已经很扎实的基础现场推理
5. **跨模块知识链每周练一次**——白纸画图，不翻资料。这是最有复利效应的练习
