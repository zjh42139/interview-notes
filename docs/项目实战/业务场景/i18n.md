# 国际化

> "国际化不只是 `$t('hello')`。当项目有 50+ 页面时，语言包如何组织？切换语言后 Element Plus 怎么同步？如何按需加载不拖慢首屏？——每个问题都是面试官想听的。"

---

## 一句话总结

国际化通过 **vue-i18n v9+ 的 Composition API** 实现多语言切换，语言包按 **模块拆分（common + 按路由分模块）** 组织，通过 **`locale.value` 动态切换** 并同步 Element Plus 语言包，用户选择持久化到 **localStorage + Pinia persist**。

---

## 核心机制

### 1. vue-i18n v9+ 初始化

```typescript
// src/i18n/index.ts
import { createI18n } from 'vue-i18n'
import zh from './locales/zh-CN'
import en from './locales/en-US'
import { getLocale, setLocale } from '@/utils/storage'

// 从 localStorage 读取用户上次选择的语言
const savedLocale = getLocale() || 'zh-CN'

const i18n = createI18n({
  legacy: false,           // 使用 Composition API 模式
  locale: savedLocale,     // 默认语言
  fallbackLocale: 'zh-CN', // 兜底语言
  messages: { 'zh-CN': zh, 'en-US': en },
  silentTranslationWarn: true,       // 生产环境关闭警告
  missingWarn: false,
  fallbackWarn: false,
})

export default i18n
```

### 2. 语言包按模块拆分

```
src/i18n/locales/
├── zh-CN/
│   ├── index.ts        # 聚合所有模块
│   ├── common.json      # 通用文案（按钮、提示、表单）
│   ├── login.json       # 登录页
│   ├── dashboard.json   # 仪表盘
│   └── user.json        # 用户管理
└── en-US/
    ├── index.ts
    ├── common.json
    ├── login.json
    ├── dashboard.json
    └── user.json
```

```typescript
// src/i18n/locales/zh-CN/index.ts
import common from './common.json'
import login from './login.json'
import dashboard from './dashboard.json'
import user from './user.json'

export default {
  common,
  login,
  dashboard,
  user,
}
```

组件中使用时带模块前缀：

```vue
<template>
  <span>{{ $t('common.save') }}</span>        <!-- 通用文案 -->
  <span>{{ $t('login.welcome') }}</span>      <!-- 登录模块文案 -->
  <span>{{ $t('user.pleaseSelect') }}</span>  <!-- 用户模块文案 -->
</template>
```

### 3. 动态切换语言 + Element Plus 同步

```typescript
// src/hooks/useLocale.ts
import { useI18n } from 'vue-i18n'
import zhCn from 'element-plus/dist/locale/zh-cn.mjs'
import en from 'element-plus/dist/locale/en.mjs'
import { ElMessage } from 'element-plus'

const elLocales: Record<string, any> = {
  'zh-CN': zhCn,
  'en-US': en,
}
const elLocale = ref(en)   // Element Plus 当前语言包

export function useLocale() {
  const { locale, t } = useI18n()

  function switchLocale(lang: 'zh-CN' | 'en-US') {
    locale.value = lang                     // 切换 vue-i18n
    elLocale.value = elLocales[lang]        // 切换 Element Plus
    setLocale(lang)                          // 持久化到 localStorage
    ElMessage.success(t('common.languageChanged'))
  }

  return { locale, elLocale, switchLocale, t }
}
```

```vue
<!-- App.vue -->
<template>
  <el-config-provider :locale="elLocale">
    <router-view />
  </el-config-provider>
</template>
```

### 4. Lazy Loading 语言包

按路由懒加载，减少首屏体积：

```typescript
// src/i18n/index.ts（增强版）
const loadedLanguages = new Set<string>()

export async function loadLocaleMessages(locale: string, module: string): Promise<void> {
  const key = `${locale}_${module}`
  if (loadedLanguages.has(key)) return   // 已加载过，跳过

  // 动态 import，此处由 Vite 分包
  const messages = await import(`./locales/${locale}/${module}.json`)
  i18n.global.mergeLocaleMessage(locale, { [module]: messages.default })
  loadedLanguages.add(key)
}
```

在路由守卫中按需加载：

```typescript
// src/router/guard.ts
router.beforeEach(async (to) => {
  const module = to.meta.i18nModule as string
  if (module) {
    await loadLocaleMessages(i18n.global.locale.value, module)
  }
})
```

## 深度拓展

### 追问 1：日期格式如何处理多语言？

使用 dayjs 的 locale 支持，切换语言时同步更新：

```typescript
// src/utils/date.ts
import dayjs from 'dayjs'
import 'dayjs/locale/zh-cn'
import 'dayjs/locale/en'
import { watch } from 'vue'
import { useI18n } from 'vue-i18n'

const { locale } = useI18n()

watch(locale, (lang) => {
  dayjs.locale(lang.toLowerCase())   // 'zh-cn' / 'en'
}, { immediate: true })

// 使用 dayjs().format('LL') 自动按当前语言输出格式
```

### 追问 2：数字、货币格式的多语言

```typescript
// 使用 Intl.NumberFormat 原生 API
function formatCurrency(amount: number, locale: string, currency: string): string {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,          // 'CNY' / 'USD'
  }).format(amount)
}

// 基于当前 locale 调用
const { locale } = useI18n()
formatCurrency(12345.67, locale.value, locale.value === 'zh-CN' ? 'CNY' : 'USD')
// 输出：¥12,345.67（中文） / $12,345.67（英文）
```

### 追问 3：富文本中的多语言

富文本（如文章、公告）一般不通过 vue-i18n 翻译，而是后端直接返回对应语言的内容：

```typescript
// 请求时带 Accept-Language 头
const { data } = await http.get('/api/articles', {
  headers: { 'Accept-Language': i18n.global.locale.value },
})
// 后端返回对应语言的文章内容，前端直接 v-html 渲染
```

## 项目实战

### 后台管理系统完整国际化方案

```
规划阶段：
1. 确定支持的语言（zh-CN, en-US）
2. 统计现有文案量，确认翻译工作量
3. 和产品确认翻译由谁提供（前端自己翻译 or 专业翻译团队）

开发阶段：
4. 搭建 vue-i18n 框架，common 模块先行
5. 开发时只用中文 key，写脚本自动提取到 JSON
6. 路由配置中加入 i18nModule meta，实现按路由懒加载

发布阶段：
7. 构建时处理：en-US 的 JSON 单独打包，可选下载
8. 监控：统计各语言使用率，作为后续优化依据
```

## 易错点

1. **切换语言后模板不刷新**：确保使用了 `ref` / `reactive` 包裹的数据或者直接使用 `$t()`，不要在 `data()` 中赋值一次后就不再更新。vue-i18n v9+ 的 Composition API 模式下，`t()` 返回的是响应式值。

2. **Element Plus 语言未同步**：只切换了 `locale.value` 但没有给 `<el-config-provider>` 传新的 locale，导致 Element Plus 组件（分页、日期选择器等）仍然是旧语言。

3. **语言包 key 冲突**：不同模块下使用了相同的 key。解决：按模块命名空间隔离（`common.xxx`、`user.xxx`）。

4. **Lazy Loading 时 JSON 文件路径错误**：Vite 的 `import()` 只支持以 `/` `./` `../` 开头的静态路径。直接拼接变量（如 `` import(`./${module}.json`) ``）Vite 无法静态分析，需要通过 `import.meta.glob` 或约定好的路径前缀处理。
